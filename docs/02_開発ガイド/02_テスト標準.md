## テストの種類と役割

ShiftWithプロジェクトでは以下の5種類のテストを実施します：

1. **単体テスト**:
   - 個別の関数・コンポーネントの機能テスト
   - Jest + React Testing Libraryを使用
   - ファイル名: `*.unit.test.(ts|tsx)`

2. **機能内結合テスト**:
   - 一つの機能内でのコンポーネント間連携テスト
   - Jest + React Testing Libraryを使用
   - ファイル名: `*.integration.test.(ts|tsx)`

3. **機能間結合テスト**:
   - 複数の機能間のコンポーネント連携テスト
   - Cypressを使用
   - ファイル名: `*.cy.ts`

4. **統合テスト**:
   - 複数機能間の連携による全体フローテスト
   - Cypressを使用
   - ファイル名: `*.system.cy.ts`

5. **E2Eテスト**:
   - エンドツーエンドのユーザーフロー全体テスト
   - Playwrightを使用
   - ファイル名: `*.e2e.test.ts`

## テスト実装ガイドライン

### 1. 単体テスト

```typescript
// src/components/common/atoms/Button/test/unit/Button.unit.test.tsx
import { render, screen, fireEvent } from '@testing-library/react';
import { Button } from '../../index';

describe('Button', () => {
  it('クリック時にonClickが呼ばれる', () => {
    const onClick = jest.fn();
    render(<Button onClick={onClick}>クリック</Button>);
    
    fireEvent.click(screen.getByText('クリック'));
    expect(onClick).toHaveBeenCalledTimes(1);
  });

  it('disabled時はクリックイベントが発生しない', () => {
    const onClick = jest.fn();
    render(<Button disabled onClick={onClick}>クリック</Button>);
    
    fireEvent.click(screen.getByText('クリック'));
    expect(onClick).not.toHaveBeenCalled();
  });
});
```

### 2. 機能内結合テスト

```typescript
// src/components/features/materials/test/integration/MaterialList.integration.test.tsx
import { render, screen, waitFor } from '@testing-library/react';
import { MaterialList } from '../../index';
import { useMaterials } from '@/hooks/useMaterials';

jest.mock('@/hooks/useMaterials');

describe('MaterialList Integration', () => {
  it('教材一覧が正しく表示される', async () => {
    const mockMaterials = [
      { id: '1', title: '教材1' },
      { id: '2', title: '教材2' }
    ];
    
    (useMaterials as jest.Mock).mockReturnValue({
      materials: mockMaterials,
      isLoading: false
    });

    render(<MaterialList />);
    
    await waitFor(() => {
      expect(screen.getByText('教材1')).toBeInTheDocument();
      expect(screen.getByText('教材2')).toBeInTheDocument();
    });
  });
});
```

### 3. 機能間結合テスト

```typescript
// cypress/e2e/integration/materials/creation.cy.ts
describe('教材作成フロー', () => {
  beforeEach(() => {
    cy.login();
    cy.visit('/materials/create');
  });

  it('教材を作成してポイントが付与される', () => {
    // 教材作成
    cy.get('[data-testid="title-input"]').type('新しい教材');
    cy.get('[data-testid="content-input"]').type('教材の内容');
    cy.get('[data-testid="submit-button"]').click();

    // 作成完了確認
    cy.url().should('include', '/materials');
    cy.contains('教材を作成しました');

    // ポイント付与確認
    cy.get('[data-testid="user-points"]')
      .should('include.text', '100');
  });
});
```

### 4. 統合テスト

```typescript
// cypress/e2e/system/giver-score.cy.ts
describe('ギバースコアシステム', () => {
  it('教材作成から診断結果反映までの一連の流れ', () => {
    // ログイン
    cy.login();

    // 教材作成
    cy.createMaterial('テスト教材');

    // ギバー診断実施
    cy.completeGiverDiagnostics();

    // スコア反映確認
    cy.get('[data-testid="giver-score"]')
      .should('exist')
      .and('not.equal', '0');

    // ダッシュボード確認
    cy.visit('/dashboard');
    cy.get('[data-testid="activity-history"]')
      .should('contain', 'テスト教材を作成');
  });
});
```

### 5. E2Eテスト

```typescript
// tests/e2e/auth.e2e.test.ts
import { test, expect } from '@playwright/test';

test('新規ユーザー登録フロー', async ({ page }) => {
  // トップページアクセス
  await page.goto('/');
  
  // 新規登録
  await page.click('[data-testid="signup-button"]');

  // 診断完了
  await completeDiagnostics(page);

  // ダッシュボードへ遷移
  await expect(page).toHaveURL('/dashboard');
});
```

## テストカバレッジ要件

### カバレッジ目標

1. **単体テスト**:
   - コンポーネント: 90%以上
   - ユーティリティ関数: 100%
   - カスタムフック: 90%以上

2. **機能内結合テスト**:
   - 主要機能フロー: 80%以上
   - エラーケース: 90%以上

3. **機能間結合テスト**:
   - コア機能連携: 70%以上
   - 主要ユーザーフロー: 80%以上

4. **統合テスト**:
   - 重要ビジネスフロー: 90%以上
   - エッジケース: 60%以上

5. **E2Eテスト**:
   - クリティカルパス: 100%
   - 主要ユーザージャーニー: 80%以上

### 優先度の定義

1. **P0** (必須):
   - 認証関連
   - データ保存・更新
   - 重要なユーザーフロー

2. **P1** (重要):
   - UI表示・インタラクション
   - エラーハンドリング
   - パフォーマンス要件

3. **P2** (望ましい):
   - エッジケース
   - 補助的機能
   - UX改善項目

## テスト実行環境

### 1. ローカル開発環境

```bash
# 単体・機能内結合テスト
npm run test        # 全テスト実行
npm run test:watch  # 監視モード
npm run test:coverage  # カバレッジレポート

# 機能間結合・統合テスト
npm run cypress     # Cypress起動
npm run cypress:run # ヘッドレス実行

# E2Eテスト
npm run test:e2e   # Playwright実行
```

### 2. CI環境

GitHub Actionsで以下を実行:

1. プルリクエスト時:
   - 単体テスト
   - 機能内結合テスト
   - カバレッジレポート

2. メインブランチマージ時:
   - 全テスト実行
   - E2Eテスト
   - パフォーマンステスト

## ベストプラクティス

1. **テストの独立性**:
   - テスト間の依存を避ける
   - テストデータは各テストで初期化

2. **テストの可読性**:
   - 明確な命名規則
   - テストケースの目的を明記
   - データセットアップを分離

3. **テストの保守性**:
   - DRYな実装を避ける
   - ヘルパー関数の活用
   - テストユーティリティの共有

4. **テストデータ管理**:
   - フィクスチャーの活用
   - テストデータの一元管理
   - 環境依存の最小化

## モック・スタブの使用指針

1. **外部依存のモック**:
   - APIコール
   - データベースアクセス
   - 外部サービス

2. **内部依存のモック**:
   - 時間依存の処理
   - ランダム値
   - 環境変数

例:
```typescript
// APIコールのモック
jest.mock('@/services/api', () => ({
  fetchMaterials: jest.fn().mockResolvedValue([
    { id: '1', title: 'テスト教材' }
  ])
}));

// 時間依存処理のモック
jest.spyOn(Date, 'now').mockImplementation(() => 1234567890);
```

## エラーケースのテスト

1. **ネットワークエラー**:
   - タイムアウト
   - 接続エラー
   - レスポンスエラー

2. **バリデーションエラー**:
   - 不正な入力値
   - 必須項目の欠落
   - 型の不一致

3. **状態エラー**:
   - 未認証アクセス
   - 権限エラー
   - データ不整合

例:
```typescript
it('ネットワークエラー時のエラーメッセージ表示', async () => {
  const error = new Error('Network Error');
  (fetchMaterials as jest.Mock).mockRejectedValue(error);

  render(<MaterialList />);
  
  await waitFor(() => {
    expect(screen.getByText('データの取得に失敗しました')).toBeInTheDocument();
  });
});
```

## パフォーマンステスト

1. **レンダリング性能**:
   - 初期表示時間
   - 再レンダリング時間
   - メモリ使用量

2. **インタラクション性能**:
   - クリックレスポンス
   - スクロール性能
   - アニメーション滑らかさ

3. **データ処理性能**:
   - 大量データの処理時間
   - メモリ効率
   - キャッシュ効果

例:
```typescript
import { measureRenderTime } from '@/utils/performance';

it('1000件のアイテム表示で500ms以内にレンダリング', async () => {
  const items = Array.from({ length: 1000 }, (_, i) => ({
    id: `${i}`,
    title: `Item ${i}`
  }));

  const renderTime = await measureRenderTime(() => {
    render(<VirtualList items={items} />);
  });

  expect(renderTime).toBeLessThan(500);
});
```
