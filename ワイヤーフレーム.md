## 4. ユーザー心理を考慮した設計のポイント

### 4.1 診断テスト設計のポイント

1. **段階的な質問提示**
   - 一度に全ての質問を表示せず、1問ずつ表示することでユーザーの認知負荷を軽減
   - 進捗バーで全体の進行状況を明示し、完了までの見通しを提供

2. **質問の工夫**
   - 回答の選択肢に具体的な例や説明を付加し、理解しやすく
   - 極端な選択肢から中立的な選択肢まで、幅広い回答の余地を提供
   - 質問文は簡潔で明確な表現を使用

3. **心理的安全性の確保**
   - 「正解」や「望ましい回答」があるような印象を与えない
   - 結果は「良い/悪い」ではなく「特性の違い」として表現
   - 回答は後から変更可能であることを明示

### 4.2 結果表示画面設計のポイント

1. **ポジティブフレーミング**
   - どのタイプでも肯定的な表現を使用
   - 「テイカー」のような否定的に捉えられる可能性のある言葉は避け、「学習者型」などの表現を使用
   - 全てのタイプに「強み」を明示

2. **成長マインドセットの促進**
   - 特性は固定的ではなく、行動によって変化することを強調
   - 具体的な成長の道筋と次のアクションを明確に提示
   - 小さな一歩から始められる簡単なアクションを推奨

3. **自己効力感の向上**
   - 詳細なスコア表示で自己理解を深める機会を提供
   - 具体的な行動提案により「何をすべきか」が明確
   - 視覚的な表現（グラフ、アイコン）で直感的に理解しやすく

4. **社会的証明の活用**
   - 他のユーザーとの比較により位置づけを理解しやすく
   - コミュニティへの貢献価値を強調
   - 「教えることで学ぶ」という科学的根拠に基づいた学習法の有効性を伝える

### 4.3 UXフローの最適化ポイント

1. **スムーズな誘導**
   - 診断結果から次のアクションへの明確な導線
   - 「ホーム画面へ」と「具体的アクション」の両方の選択肢を用意
   - ユーザーの現在のレベルに合わせた適切な難易度の行動を提案

2. **パーソナライズ体験**
   - ユーザータイプに応じて異なる結果表示とアクション提案
   - ギバースコアに基づいた詳細な特性分析
   - 将来的には学習履歴と組み合わせたさらに個別化された提案が可能

3. **共有機能の活用**
   - 診断結果の共有ボタンで口コミ拡散を促進
   - プロフィールへの結果保存で自己アイデンティティ形成を支援
   - コミュニティでの位置づけを認識させるきっかけに

## 3. デザイン要素の説明

### 3.1 レイアウトの基本設計

1. **プライオリティゾーニング**
   - 最上部: ギバースコアとユーザーの現在の状態（ストリーク、タイプ）
   - 中央部: アクション誘導と推奨コンテンツ
   - 下部: 補足情報（バッジ、活動履歴）

2. **視覚的階層**
   - 大きいカード: 最も重要な情報（ギバースコア、今日やるべきこと）
   - 中サイズカード: 推奨コンテンツと最近の活動
   - 小カード: 補足情報（Tips、バッジ）

3. **レスポンシブ対応**
   - デスクトップ: 多列レイアウトで情報密度を高く
   - モバイル: 単列スクロールでシンプルに、詳細は別画面に分離

### 3.2 主要コンポーネントの意図

1. **ギバースコアカード**
   - 目的: ユーザーの現在の状態を一目で把握し、成長を実感させる
   - 特徴:
     - スコアの視覚的表現（プログレスバー）
     - 前回からの変化を明示（+5ポイント）
     - タイプ表示で自己認識を強化
     - 次のレベルまでの残りを表示し目標を意識させる

2. **今日やるべきことカード**
   - 目的: 明確な行動指針を提供し、日々のアクションを促進
   - 特徴:
     - チェックリスト形式で達成感を得やすく
     - 各項目に具体的なアクションリンクを提供
     - 優先度順に表示（学習→フィードバック→ポイント獲得）

3. **おすすめ教材セクション**
   - 目的: 学習コンテンツへの導線を確保し、関心を引くコンテンツを提示
   - 特徴:
     - ビジュアル重視（サムネイル表示）
     - グリッドレイアウトで複数コンテンツを一覧可能
     - パーソナライズされた推薦（ユーザーの学習履歴に基づく）

4. **最近の活動セクション**
   - 目的: 継続的な活動を振り返り、進捗を実感させる
   - 特徴:
     - 時系列表示で最新の活動を強調
     - 多様な活動（学習・作成・バッジ獲得）を表示
     - 活動の詳細へのリンクを提供

5. **ギバーTipsセクション**
   - 目的: 「教えることで学ぶ」という核心コンセプトを常に意識させる
   - 特徴:
     - 科学的根拠に基づく情報提供
     - 具体的なアクション提案
     - 簡潔なメッセージングで負担を感じさせない

### 3.3 ユーザー心理を考慮した設計ポイント

1. **動機付けの工夫**
   - 進捗の可視化（スコア、バッジ、ストリーク）
   - 小さな成功体験の積み重ね（チェックリスト、日次目標）
   - 成長の実感（先週比、レベルアップ）
   - 社会的証明（コミュニティハイライト）

2. **行動障壁の低減**
   - 明確なアクションボタン（「教材を作成する」など）
   - 複雑なタスクの分割（「簡単な教材」の強調）
   - 選択肢の適切な限定（おすすめ教材を厳選）
   - ワンクリックアクセス（直接リンクの多用）

3. **習慣形成の支援**
   - 日次チェックリストで定期的なアクション促進
   - ストリーク表示で連続行動を奨励
   - デイリーログインポイントで定期訪問を促進
   - 「今日の」という表現で即時行動を喚起

4. **ユーザー成長のサポート**
   - レベル設計による段階的な成長パス
   - 特典（機能強化、割引）によるインセンティブ
   - 能力別スコアで具体的な成長領域を提示
   - 成長Tipsで学習効果を高める情報提供

### 3.4 モバイルとデスクトップの最適化差異

1. **情報密度の調整**
   - デスクトップ: 複数カラム、詳細情報の同時表示
   - モバイル: 単一カラム、詳細情報は別画面に分離

2. **操作性の最適化**
   - デスクトップ: ホバー効果、ツールチップの活用
   - モバイル: タップターゲットの拡大、ボトムナビゲーション

3. **視覚要素の調整**
   - デスクトップ: 大きなグラフ、詳細な

## 3. 教材作成画面の設計ポイント

### 3.1 作成ハードルを下げるための工夫

1. **段階的な選択肢の提供**
   - クイック投稿から詳細教材まで、難易度別の選択肢
   - 作成時間の目安を明示（1〜2分、5〜15分、etc）
   - 初心者向けのガイダンス（「初めての方はクイック投稿から」）

2. **フォーム設計の最適化**
   - 必須項目を最小限に（*マークで明示）
   - 文字数制限の表示（30文字まで、300文字までなど）
   - ドロップダウンでの選択肢提供（カテゴリ、学習時間など）

3. **段階的なフォーム進行**
   - 複数ステップに分割（基本情報→コンテンツ→確認と公開）
   - 進捗表示（1/3, 2/3, 3/3）
   - いつでも下書き保存可能

### 3.2 モバイルでの入力効率化

1. **入力補助ツールの提供**
   - シンプルな書式ボタン（B, I, 見出し、リスト）
   - 画像追加ボタンの配置
   - セクション追加機能

2. **UI要素の最適化**
   - タップしやすいボタンサイズ
   - スクロールの最小化
   - プレビュー機能

3. **自動保存と下書き機能**
   - 定期的な自動保存
   - いつでも下書き保存できるボタン
   - 下書きの続きから編集再開

### 3.3 ユーザーを教材作成に導く工夫

1. **インセンティブの明示**
   - ギバースコア獲得量の表示
   - ポイント獲得量の表示
   - 公開後の成功体験（🎉演出）

2. **品質向上サポート**
   - クオリティチェックリスト
   - 改善提案（画像追加、練習問題追加など）
   - 全画面プレビュー機能

3. **次のステップへの誘導**
   - 公開成功後の選択肢提示
   - 共有機能によるソーシャル拡散促進
   - 自分の教材統計確認への導線

### 3.4 教材管理・モチベーション維持の工夫

1. **成果の可視化**
   - 閲覧数・評価の数値表示
   - グラフによる推移表示
   - ポジティブなフィードバック表示

2. **改善サイクルの促進**
   - 評価・コメントのフィードバック表示
   - 改善点の示唆
   - 編集機能へのアクセス

3. **継続作成の動機付け**
   - 統計データによる効果実感
   - 新規教材作成への誘導
   - ギバースコアとの連動
